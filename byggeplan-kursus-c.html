Kursus C-systemet — redesign & byg

Åbn denne fil i en ny chat. Find siden du vil lave, tryk Kopiér prompt, indsæt den i chatten — så ved den præcis hvad den skal gøre.

Sådan bruger du planen

  • Start en frisk chat til hver side (kvaliteten falder i lange tråde).
  • Har chatten adgang til denne fil (i projektet), så nævn den. Ellers indsæt bare prompten — den er selvbærende, og hukommelsen kender allerede standarden fra C1/C2.
  • Rækkefølgen nedenfor er anbefalet, men du kan tage siderne i den orden du vil.
  • To slags opgaver: RE-SKIN = siden findes med indhold i gammelt look → kun det visuelle laves om, indhold+JS urørt. BYG FRISK = siden findes ikke → bygges ny i C-designet.

1C-design-standarden

Det fælles fundament for alle C-sider. Hver prompt bygger på dette.

Token-system (accenten)

Hele accenten styres af 8 variabler i :root (5 hex + 3 rgb). Afledte glow/tint/shadow-tokens skal pege på var(--c-gold-rgb) — aldrig hardkodede tal — så et nyt niveau kun kræver de 8. Gamle accent-variabler (--teal, --indigo, --violet, --vio, --blue) omdirigeres til guld-tokens, så eksisterende CSS følger med automatisk.

Fælles fundament (uændret på alle niveauer)

  • Pergament #e8e2d3 · creme-kort #fbf7ee · ink #20312a · kant #cfc6b0
  • Fonte: Lora (serif overskrifter) + Outfit (brødtekst)
  • Sidebar: beige creme-panel #ddd3ba (--sb-bg) med runde 20px hjørner — porteres uændret fra C1
  • Cinematic intro: mørk skov rgba(47,106,74)→#0a1410 + guld-motes, grønt logo #2F5D45 + guld niveau-mærke

Farve-variation (VIGTIGT — ikke alt skal være guld)

  • Guld = primær/struktur: hero, modul-numre, progress, knapper, aktiv navigation
  • Grøn = positivt: tips, gode eksempler, completion/'færdig', succes
  • Rav = advarsler / 'pas på'
  • Rød = fejl / dårlige eksempler

Faste regler

  • Aldrig SVG i CSS content:'' — CSS kan ikke rendere SVG (viser rå kode). Brug rene glyffer ✓ / ✕ der. SVG-ikoner bruges kun i HTML.
  • Ingen emojis → rene SVG-linjeikoner (1.15em, stroke=currentColor) / nummererede guld-badges / geometriske glyffer
  • Demoer: adskil brugerens prompt (egen boks + label "Dit prompt") tydeligt fra AI'ens svar (boble + label "AI'ens svar"). Skriv "Dit prompt" — ikke "Din".
  • Behold AI-værktøjers ægte logofarver (Gemini/Copilot-blå osv.) i sidebar — de er brand-mærker
  • Brand: synligt navn altid "Akademiet AI" — aldrig "LearnAI". Frosne kodenavne urørt: learnai-dk, learnai-proxy, learnai_api_base, LEARNAI_API

Workflow (hver fil)

  • Hent altid live rå-fil fra GitHub før redigering — gæt aldrig på strukturen
  • Kirurgiske ændringer (str_replace / scriptet recolor), ikke fuld omskrivning — INDHOLD + JS byte-uændret ved re-skin
  • Validér: node --check (JS) + Python/lxml (DOM-balance)
  • Verificér visuelt med Playwright-screenshot (tving .reveal frem; dismiss intro); pixel-sample hvis view-værktøjet svigter
  • Tjek: 0 synlig "LearnAI", 0 rester af gammel accent (teal/indigo/violet), brand + frosne navne intakte
  • Lever komplet upload-klar fil via present_files

2Token-værdier pr. niveau

De præcise 8 tokens for hvert niveau (accent-progression: guld → gul → orange). Kopiér ind i den relevante prompt — de står også i hver prompt nedenfor.

Niveau--c-golddeepmidwarminkrgb

Mini-quizzer arver accenten fra kurset lige før dem (quiz-c1 = C1, quiz-c2 = C2, quiz-c3 = C3 osv.). C1 = #c2a038, C2 = #c9a040 (allerede lavet).

3Prompts pr. side

Anbefalet rækkefølge. Hver prompt er klar til at indsætte i en frisk chat.

4Åbne beslutninger (afklar først)

  • Forløbsstruktur med C4 + C5: Den nuværende journey er C1 → Q1 → C2 → Q2 → C3 → E1 → E2. Med C4/C5 skal du beslutte den nye rækkefølge (fx C1→Q1→C2→Q2→C3→Q3→C4→Q4→C5→Q5→E1→E2) og om der skal være quiz efter C3/C4/C5. Når det er besluttet, bruges "Forløbs-opdatering"-prompten nederst til at rette journey-bar + sidebar + lås-logik på ALLE C-sider ens.
  • Indhold til C4 + C5: Disse kurser har intet indhold endnu. Beslut modul-emnerne (eller lad chatten foreslå et pensum til godkendelse) FØR den bygger.
  • Verificér eksamen-c + eksamen-c-del2: De findes nu på GitHub (modsat tidligere note). Bekræft de er komplette før re-skin — ellers bygges manglende dele først.
  • C-belønnings-flow (til diplomet): eksamen-c-del2 skriver ikke coins/badge til Firestore endnu (kun UI-state). Beslut — som ved D — at del2 sætter et passed-flag, og at diplomsiden udsteder belønningen (150 coins + Sølv-badge C + arrayUnion('C') + bevis) ét sted med dedup, så der aldrig belønnes dobbelt.
  • quiz-c2.html er kun ~428 linjer (mod quiz-c1's ~1283) — tjek om den er færdig eller en stub, før re-skin.
  • Cheat sheets røres ikke under C-redesign (jsPDF-downloaden) — bevar koden som den er; hele cheat-sheet-designet laves i "Mit Bibliotek"-chatten senere.
